In this experiment, I added a input filed which could works as a filter.

Now this list has only 8 items, but if this list grows longer, it would be harder to find the item you want.

By this filter, user could quickly find the item by input some key words, and the result would change with the input or backspace.


In the html, I use ul to contain the data. input to hold the user's input. Give specified class to each tag.

<div class="" id="test-list"> <input type="text" class="fuzzy-search form-control" placeholder="Search..." /> <ul class="list"> <li><p class="name">Kigo Kitchen</p></li> <li><p class="name">On The Go</p></li> <li><p class="name">Popeyes Louisiana Kitchen</p></li> <li><p class="name">Starbucks</p></li> <li><p class="name">Sweet Tomatoes Pizza</p></li> <li><p class="name">Taco Bell Express</p></li> <li><p class="name">The West End</p></li> <li><p class="name">UBurger</p></li> </ul> </div>

Then use list.js and list.fuzzysearch.js to achieve the aim to filter the data by input.

Active the plug-in:

<script> var monkeyList = new List('test-list', { valueNames: ['name'], plugins: [ListFuzzySearch()] }); </script>

The default search will conduct a time efficient search for an exact match in the content searched, while this one will render results depending on if they are included anywhere in the content.
